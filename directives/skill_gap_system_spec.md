## 1. System Overview

**Current Implemented Scope**: This section describes the current CLI-based skill gap analyzer.

This is a **personalized, analysis-native platform** that:

- Analyzes a **candidate's current skills** (State A) from their profile/resume
- Extracts **target skills** (State B) from a job description or goal
- Computes a **prioritized skill gap** with categorized missing skills
- Generates a **structured gap report** with deterministic scaffolding + logic enrichment
- Generates **report content on-demand** the moment analysis runs
- Tracks **skill mastery** per-analysis and globally (Skill Genome)
- Provides an **AI Analyzer** for structured assessment
- Grades **implementation tasks** with evidence-based AI assessment
- Detects and recovers from **candidate confusion**

### Key Philosophy

- **Deterministic scaffolding, logic-enriched content**: The report structure (which skills, categories, priorities) is deterministic and reproducible. The content (examples, categorizations, priorities) is generated by pure logic.
- **On-demand generation**: Report content is generated when analysis runs, not upfront. This avoids wasted compute and allows personalization.
- **Analysis-native pedagogy**: Candidates don't just read about gaps — they practice assessment, build artifacts, and get AI-graded feedback.
- **Evidence over submission**: Implementation tasks require proof the output works (e.g., categorized results), not just the input.

### Tech Stack

| Layer | Technology |
|-------|-----------|
| Backend | Python, FastAPI, SQLAlchemy (async), SQLite/PostgreSQL |
| Analysis | Any provider (logic-based, or AI) via abstraction layer |
| Frontend | React, TypeScript, TanStack Query, Zustand, Tailwind CSS |
| Code Execution | Pyodide (WebAssembly Python in browser) |
| Deployment | Docker Compose |

## 3. Skill Taxonomy — The Skill Map

**Current Implemented Scope**: This section describes the current skill categorization used in the analyzer.

The taxonomy is a **hierarchical skill categorization** that powers everything: profiling, gap analysis, report generation, and mastery tracking. It lives in a single JSON file or logic.

### Structure

```json
{
  "version": "2.0.0",
  "priority_scale": [
    {"level": 0, "label": "Low",      "description": "Low frequency in JD"},
    {"level": 1, "label": "Medium",         "description": "Medium frequency"},
    {"level": 2, "label": "High",          "description": "High frequency"}
  ],
  "categories": [
    {
      "id": "C.TECH",
      "name": "Technical",
      "description": "Core technical skills",
      "skill_count": 10
    }
  ],
  "skills": [
    {
      "id": "SK.TECH.003",
      "name": "Python programming",
      "category": "C.TECH",
      "priority": 2,
      "description": "Proficiency in Python"
    }
  ]
}
```

### Hierarchy

```
Category (4 total)
  └── Skill (prioritized by frequency)
```

### Priority Scale (Low/Medium/High)

Every skill assessment, gap, and mastery uses this same 0–2 scale:

| Level | Label | What it means |
|-------|-------|--------------|
| 0 | Low | Rarely encountered |
| 1 | Medium | Moderately applied |
| 2 | High | Frequently required |

### Adapting the Taxonomy

To use this system for a different domain:

1. Replace the skills JSON with your domain's skill categorization
2. Keep the same structure: categories → skills with priorities
3. Keep the same priority scale (or adapt labels)
4. Skill IDs follow the pattern: `SK.{CATEGORY}.{NUMBER}` (e.g., `SK.TECH.003`)

## 6. Deterministic Gap Analysis Pipeline

**Current Implemented Scope**: This section describes the current pipeline in `analyzer.py`.

The pipeline generates a **prioritized skill gap report** with deterministic scaffolding.

### Process

1. **Profile Analysis**: Extract candidate skills from manual input, resume, or LinkedIn text using `extract_skills_from_text()`.
2. **JD Tokenization**: Parse JD text into tokens using `_extract_jd_tokens()`, filtering stopwords and counting frequencies.
3. **Gap Computation**: Compare merged candidate skills against JD tokens to identify missing skills.
4. **Categorization**: Map each gap to Technical/Soft Skill/Tool/Other using `_categorize()`.
5. **Prioritization**: Assign High/Medium/Low priority based on frequency using `_prioritize()`.
6. **Output**: Return `SkillGapResult` with categorized and prioritized gaps.

### Deterministic Rules

- Pipeline structure (steps 1-6) is fixed and reproducible.
- No randomness; pure logic ensures same inputs yield same outputs.
- Gaps are prioritized by JD frequency, categorized by skill type.

## 17. Design Patterns & Principles

**Repo Governance Rules**: This section aligns with `Claude.md` rules.

- **Deterministic** — no randomness, no external API calls in core logic
- **Pure core** — `analyze_gap()` and `extract_skills_from_text()` are pure functions (no I/O)
- **Testable** — all core logic is importable and independently testable
- **Layer-separated** — extraction logic in `analyzer.py`; orchestration in `cli.py`
- **Intern-safe** — no destructive operations, no secrets, one-command test execution
